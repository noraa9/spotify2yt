# ADR-001: Initial Architecture

## Status

Accepted

## Context

Проект представляет собой CLI-приложение для переноса плейлистов между музыкальными сервисами.

На раннем этапе важно определить базовую архитектуру, которая позволит постепенно расширять функциональность без переписывания существующего кода.

## Decision

Были приняты следующие решения:

- Выделить отдельный `TransferService`, отвечающий только за процесс переноса.
- Работать с музыкальными сервисами через абстракцию `Provider`.
- Использовать общие доменные модели (`Track`, `Playlist`), независимые от конкретных API.
- Разрабатывать проект итеративно: сначала MVP, затем постепенные улучшения.

## Consequences

Преимущества выбранного подхода:

- Добавление нового музыкального сервиса требует реализации нового Provider без изменения бизнес-логики.
- TransferService остается независимым от деталей реализации API.
- Проект проще поддерживать и расширять.

# ADR-002: Use Provider abstraction for music services

## Status

Accepted

## Date

2026-07-18

## Context

Проект должен поддерживать перенос плейлистов между музыкальными сервисами.

На первом этапе поддерживаются:

- Spotify
- YouTube Music

В будущем возможно добавление других сервисов, например:

- Apple Music
- Deezer

Если TransferService будет напрямую зависеть от конкретных API, то при добавлении нового сервиса придется изменять его код.

Это нарушает принцип Open/Closed и увеличивает связанность компонентов.

## Decision

Все музыкальные сервисы взаимодействуют с приложением через абстракцию `Provider`.

Каждый Provider самостоятельно:

- выполняет авторизацию;
- взаимодействует с API платформы;
- преобразует ответы API в доменные модели приложения.

TransferService работает только с интерфейсом `Provider` и не знает о деталях реализации конкретных платформ.

## Consequences

### Преимущества

- TransferService остается независимым от Spotify и YouTube Music.
- Добавление нового сервиса требует только реализации нового Provider.
- Доменные модели остаются едиными для всех платформ.
- Код становится проще тестировать.

### Недостатки

- Требуется дополнительный слой абстракции.
- Каждая платформа должна реализовать интерфейс Provider.
